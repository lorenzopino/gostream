### Stub file format (gostream-real)
```json
{"imdb":"tt32141377","magnet":"magnet:?xt=urn:btih:HASH...","size":20804790244,"url":"http://127.0.0.1:8090/stream?link=HASH&index=1&play"}
```

### Problema FUSE + Jellyfin scan
**Sintomo**: scan Jellyfin si blocca al ~4%, solo pochi film visibili.
**Root cause**: Jellyfin esegue FFprobe su ogni `.mkv` FUSE virtuale → GoStorm avvia download torrent per servire i byte → 60-90 sec per file → 5+ ore per scansionare 694 file.
**Non è un vero blocco**: la scan procede ma lentissima (1 item/90s).

### Soluzione adottata: libreria .strm
Script `~/MediaCenter/scripts/generate-strm-library.py` genera `.strm` + `.nfo` da tutti gli stub.
- `.strm` contiene: `http://host.docker.internal:8090/stream?link=HASH&index=1&play`
- `.nfo` contiene IMDB ID per match diretto in Jellyfin
- Output: `~/MediaCenter/strm-library/movies/` e `~/MediaCenter/strm-library/tv/`
- Volume Jellyfin: ` [truncated]
